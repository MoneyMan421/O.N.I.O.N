# Contributing

Thank you for your interest in contributing to O.N.I.O.N!

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b feature/my-change`)
3. **Write code** following the existing style and conventions
4. **Run tests** to verify your changes do not break existing behavior
5. **Submit a PR** with a clear description of what was changed and why

Please read the [Code of Conduct](CODE_OF_CONDUCT.md) before contributing.  
For security-related issues, see [SECURITY.md](SECURITY.md) — do not open public issues for vulnerabilities.
